# Contracts: Window & Viewer Stability (006)

This desktop feature has no HTTP/API contract. Its contracts are local persistence and subprocess invocation.

## Contract 1 — Window preferences JSON (NEW, FR-008 / SC-005)

**Path**: `~/.config/rust-feh/window-prefs.json`

**Writer**: `save_window_prefs(&WindowPreferences)` in `src/ui_logic.rs`

**Reader**: `load_window_prefs()` in `src/ui_logic.rs`

**JSON schema (v1)**:

```json
{
  "version": 1,
  "preset": "Default",
  "resizable": true
}
```

### Fields

| Field | Required | Type | Valid values | Default |
|-------|----------|------|--------------|---------|
| `version` | yes | integer | `1` | `1` |
| `preset` | yes | string enum | `"Compact"`, `"Default"`, `"Large"` | `"Default"` |
| `resizable` | yes | boolean | `true`, `false` | `true` |

### Behavioral contract

- Missing file MUST NOT be an error; return defaults.
- Corrupt / unparseable file MUST NOT crash; warn to stderr and return defaults.
- Saves SHOULD use temp-file + rename, matching `save_launch_list`, so a partial write does not leave a truncated final file under normal filesystem semantics.
- No egui/eframe types may appear in this contract or in `WindowPreferences`; this is core domain state.
- The app must persist on user changes to preset/resizable choice, not wait for a graceful app exit.

### Compatibility

- Unknown future `version` handling is not implemented in v1. Future migrations should preserve the fallback-to-default behavior for unparseable/malformed content.
- Enum serialization uses serde's default externally tagged string names. Renaming `WindowSizePreset` variants is a breaking config change and requires migration.

## Contract 2 — feh viewer launch flags (EXISTING, FR-006 / FR-007)

**Launcher**: `spawn_feh_viewer` in `src/main.rs`

Every rust-feh-launched viewer process MUST include:

```text
feh --geometry 1280x960 --scale-down --zoom max --filelist <list-path> --start-at <image-path>
```

### Behavioral contract

- `--geometry 1280x960` prevents feh windows from shrinking to tiny-image dimensions.
- `--scale-down --zoom max` lets small/tiny images be inspectable while keeping normal images within the fixed viewer window.
- `<list-path>` is generated by rust-feh (`feh_filelist_temp_path` or `feh_entry_filelist_path`) and contains one image path per line.
- `<image-path>` must be a member of the filelist.

### Negative cases

- If `feh` is unavailable, rust-feh must report the existing missing-feh status and not panic.
- If the filelist cannot be written, rust-feh must report failure and not spawn feh.
